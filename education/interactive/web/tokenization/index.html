What is Tokenization?

Tokenization (lexical analysis) is the first phase of compilation. The lexer reads source code character-by-character and groups them into meaningful units called tokens.

Why Tokenization?

  • Simplifies parsing by breaking text into meaningful chunks
  • Removes whitespace and comments
  • Identifies keywords, operators, identifiers, and literals
  • Reports lexical errors (invalid characters, malformed numbers)

Key Concepts

üéØ Maximal Munch

The lexer matches the longest possible token. For example, == is one token, not two = tokens.

‚ö° Lookahead

The lexer sometimes needs to peek ahead to determine token boundaries (e.g., distinguishing = from ==).

üî§ Keywords vs Identifiers

Words like fun and let are keywords, not identifiers, even though they look like identifiers.

üö´ Error Recovery

When an invalid character is encountered, the lexer reports an error and continues to find more tokens.
